### Version 1:
یک اسکریپت پاورشل بنام backuper.ps1 بنویس که فرآیند کپی یکسری فایل/دایرکتوری را در محل های مشخص شده انجام دهد. در واقع نوعی Backuper محلی است. این اسکریپت باید به شکل زیر عمل کند:  
قدم اول: ابتدا دو لیست بنام های source_dir و target_dir را بسازد که در مقادیر آنها بصورت زیر میباشند:  
1. source_dir => حاوی یکسری Path دایرکتوری یا فایل است که به عنوان مبدا هستند و باید در دایرکتوری های موجود در target_dir کپی شوند.  
2. target_dir => حاوی یکسری Path دایرکتوری است که به عنوان مقصد کپی استفاده میشوند.  
قدم دوم: سپس اسکریپت باید تمام دایرکتوری و فایل های درون source_dir را درون target_dir کپی کند.  
نکات:  
1. اگر درون source_dir دایرکتوری وجود داشت، باید آن دایرکتوری با کل محتویات آن درون تمام path های موجود درون target_dir کپی شود و اگر هم که فایل بود باید آن فایل درون تمام آدرس های درون target_dir کپی شوند.  
2. آدرس ها Path هایی که درون source_dir و target_dir وارد میشوند ممکن است دارای Space باشند بنابراین این موضوع باید توسط اسکریپت هندل و مدیریت شود.  
امکانات:  
فلگ —source: اگر اسکریپت با فلگ —source و یک Path استفاده شد باید Path وارد شده به لیست source_dir اضافه شود(فایل یا دایرکتوری). مثلا اگر دستور زیر اجرا شد:  
`backuper.ps1 —source H:\Repo\Dir1` باید دایرکتوری H:\Repo\Dir1 به لیست source_dir اضافه شود.  
فلگ —target: اگر اسکریپت با فلگ —target و یک Path استفاده شد باید Path وارد شده به لیست target_dir اضافه شود(فقط دایرکتوری). مثلا اگر دستور زیر اجرا شد:  
`backuper.ps1 —target D:\Backup\` باید دایرکتوری D:\Backup به لیست target_dir اضافه شود.  
اگر هم که اسکریپت بصورت خالی اجرا شد که اسکریپت باید کار خود را انجام دهد. تمام Object های موجود در source_dir را باید درون تمام مسیر های درون target_dir کپی کند.  
مثال:    
source_dir = {“H:\File1.png“, “D:\Obsidian\”, C:\Files\Programs\”}
target_dir = {“H:\Backup1”, “D:\Backup2”, “E:\Backup3"\}  
بنابراین باید “H:\File1.png“ و  “D:\Obsidian\” و C:\Files\Programs\” درون دایرکتوری های “H:\Backup1” و  “D:\Backup2” و  “E:\Backup3"\ کپی شود.


### Version 2:
کارکرد اسکریپت درست است اما باید موارد زیر اضافه شود:  
1. اگر که کامند با فلگ —save اجرا شد تمام آدرس هایی که در —source و —target وارد شده اند در یک لیست درون اسکریپت ذخیره شوند تا در دفعه بعدی اگر اسکریپت بدون پارامتر وارد شد نیز بدرستی موارد درون source_dir درون target_dir کپی شود. مثلا اگر دستور زیر اجرا شد:  
.\backuper.ps1 -Source "H:\Files\Obsidian\Handouts" -Target "D:\Backups", "F:\Backups" —save  
باید آدرس های درون D:\Backups", "F:\Backups درون target_dir و همچنین H:\Files\Obsidian\Handout درون source_dir ذخیره شود و اگر دفعه بعدی .\backuper.ps1 اجرا شد فرآیند کپی بصورت خودکار در بین مسیر های وارد شده انجام شود.  
2. اگر اسکریپت با فلگ های —target و —source وارد شد آدرس های جدیدی که درمقابل این دو فلگ وارد شده اند باید به آدرس های قبلی که ذخیره شده اند اضافه شود.  
3. اگر هم اسکریپت با فلگ —reset-target استفاده شد، تمام آدرس های ذخیره شده درون target_dir باید حذف شوند و لیست خالی شود. اگر هم اسکریپت با —reset-source استفاده شد، تمام آدرس های ذخیره شده درون source_dir حذف شوند و لیست خالی شود.  
4. یک راهنما HELP هم برای اسکریپت بنویس که تمام فلگ ها و نحوه استفاده از برنامه را با فلگ —help نمایش دهد.  
  
بنابراین اسکریپت در دفعه اول اجرا اگر که بدون —source و —target وارد شد باید پیام Error: No source paths specified! به همراه usage, example نمایش داده شود.


### Version 3(Final: backuper.ps1):
حال قابلیت جدید را به اسکریپت اضافه کن:  
1. بصورت پیشفرض فرآیند Copy بصورتی انجام شود که اگر فایل های درون source_dir درون target_dir وجود داشتند فرآیند Skip انجام شود یعنی فایل هایی که وجود دارند دیگر نیازی نیست کپی شوند چون این اسکریپت قرار است بصورت روزانه برای بکاپ گرفتن از فایل های مهم اجرا شود و هر روز یکسری فایل جدید اضافه میشود و نه اینکه تمام فایل ها جدید باشد  
2. اگر اسکریپت با فلگ —replace اجرا شد فرآیند کپی بصورت Copy and Replace اجرا شود یعنی فایل هایی که وجود دارند نیز با فایل های source_dir بصورت Replace شوند.  
3. اگر اسکریپت با فلگ —schedule اجرا شد، اسکریپت به نحوی به Task Scheduler ویندوز اضافه شود که پس هر بار روشن شدن ویندوز و لاگین کاربر اسکریپت یکبار اجرا شود و سپس بسته شود. در واقع فقط یکبار اجرا در هنگام روشن شدن و سپس بسته شدن.  
4. این موارد جدید را در help نیز اضافه کن. ضمنا در Help مثال های استفاده یعنی Usage Example نیز اضافه کن تا کاربر دقیقا بفهمد از اسکریپت چگونه استفاده کند.
